# SentimentScope — NLP Analytics Platform

A production-grade sentiment analysis dashboard powered by a Python/Flask NLP backend and a dynamic Chart.js frontend.

---

## Project Structure

```
sentimentscope/
├── frontend/
│   ├── index.html       ← Dashboard UI
│   ├── style.css        ← Dark theme + responsive layout
│   └── script.js        ← All chart logic + API integration
│
├── backend/
│   ├── app.py           ← Flask REST API (6 endpoints)
│   ├── process_data.py  ← One-time NLP preprocessing pipeline
│   ├── requirements.txt
│   └── data/
│       ├── raw_data.csv         ← Your original dataset
│       └── processed_data.json  ← Generated by process_data.py
│
└── README.md
```

## Quick Start

### 1. Install Python dependencies

```bash
cd backend
pip install flask flask-cors
```

### 2. Run the data preprocessing pipeline (one-time)

```bash
cd backend
python process_data.py
```

This will:
- Load your raw CSV
- Clean & deduplicate rows
- Augment dataset to 10,000+ rows
- Run VADER sentiment scoring on every row
- Extract keyword tokens
- Assign regions and timestamps
- Output `data/processed_data.json`

### 3. Start the Flask backend

```bash
cd backend
python app.py
```

The API will be live at: `http://127.0.0.1:5000`

### 4. Open the frontend

Open `frontend/index.html` directly in your browser, OR serve it with a simple HTTP server:

```bash
cd frontend
python -m http.server 8080
# then visit http://localhost:8080
```

## API Endpoints

| Endpoint | Description | Query Params |
|---|---|---|
| `GET /api/health` | Server health check | — |
| `GET /api/overview` | KPI summary (total, %, trending) | `keyword`, `region`, `sentiment` |
| `GET /api/trends` | Time-series sentiment data | `period` (daily/weekly/monthly), `keyword`, `region` |
| `GET /api/keywords` | Keyword frequency breakdown | `region` |
| `GET /api/regions` | Region-wise sentiment stats | `keyword` |
| `GET /api/wordcloud` | Top words for word cloud | `keyword`, `region` |
| `GET /api/insights` | NLP-generated insights | `keyword`, `region` |
| `GET /api/anomalies` | Spike/anomaly detection | `keyword`, `region` |
| `GET /api/analyze` | Full search + filter query | `q`, `region`, `limit` |

## NLP Pipeline

```
Raw CSV
  ↓ Remove nulls, duplicates, noisy rows
  ↓ Lowercase, normalize dates, standardize regions
  ↓ Remove URLs, @mentions, hashtag symbols, punctuation
  ↓ Tokenize → remove stopwords
  ↓ VADER sentiment scoring (compound score → label)
  ↓ Feature engineering: cleaned_text, sentiment_score, sentiment_label, word_count, tokens
  ↓ Augment to 10K rows with realistic distribution
  ↓ Output: processed_data.json (loaded by Flask at startup)
```

## Data Flow

```
Browser (script.js)
    ↕  fetch()
Flask API (app.py)
    ↕  loads at startup
processed_data.json  ←  process_data.py  ←  raw_data.csv
```

- **Offline fallback:** If the Flask backend is not running, `script.js` automatically uses built-in mock data so the dashboard still renders.
- **API status pill** in the navbar shows `API live` (green) or `offline · mock data` (red).

## Deployment

### Frontend → Vercel
1. Push the `frontend/` folder to a GitHub repo
2. Import into Vercel — it deploys as a static site automatically
3. Update `API_BASE` in `script.js` to your backend URL

### Backend → Render
1. Push the `backend/` folder to GitHub
2. Create a new Web Service on Render
3. Build command: `pip install -r requirements.txt && python process_data.py`
4. Start command: `gunicorn app:app`
5. Add `gunicorn` to `requirements.txt`
